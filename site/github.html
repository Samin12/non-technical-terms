GitHub

TLDRWhere your code lives and gets shared.

Think of a shared Google Drive for code. You drop your work in, your teammate sees it, everyone always has the latest version. Now make it better: nothing is ever overwritten. Every version of every file, forever, with a record of who changed what and why.

That's GitHub. And the first thing to untangle is the name: Git and GitHub are not the same thing. Git is the save system running on your machine — the camera. GitHub is the website where projects' save histories live online — the shared album everyone can open.

GitHub is the home on the internet where your project's history lives, gets backed up, and gets shared.

Push and pull — you just did it

When you made a commit in the widget, it landed on your laptop only. GitHub was behind, and your teammate had no idea your work existed. Push sent your commits up to the shared copy. Pull brought GitHub's news down to whoever asked for it. That's the entire rhythm of collaborative building: commit → push → everyone else pulls. No emailing zip files, no "final-v7-SARA-EDITS", no overwriting each other — Git knows how the histories fit together.

It's more than storage

GitHub is also where the world's open source code lives — millions of free projects you can read, copy, and learn from. It's where pull requests happen, so strangers and teammates can propose changes safely. It's a public portfolio: people check a GitHub profile the way they check a designer's portfolio site. And it's where your AI plugs in — Claude can read a repo to understand a project, and even open pull requests there for you.

One warning before you push anything public: bots scan GitHub around the clock hunting for leaked API keys. Push a key by accident and it can be found in minutes — even if you delete it afterward, because the history keeps everything. Keys belong in a hidden env file that never gets pushed. If a key ever slips out: revoke it immediately and get a new one.

What this unlocks for you

  • A backup that survives anything — your laptop is mortal; your pushed history isn't
  • Working with anyone — humans and AI agents sharing one project without stepping on each other
  • The world's code library — almost every tool you'll ever install starts life as a GitHub repo
  • A public track record — your commits and contributions become receipts of what you can build
  • A launchpad — most hosting services deploy your site straight from your GitHub repo

Git gives you save points. GitHub puts the album somewhere safe, shared, and public enough to matter.

Your laptop is mortal. GitHub isn't.

✓ Check yourself

1. You committed a fix on your laptop, but your teammate says they don't see it. Why?

Exactly what the widget showed: a commit lives on your machine until you push it up, and others don't have it until they pull it down.

2. You're on a plane with no wifi. Can you keep working with Git?

Git is the save system on your machine; GitHub is the shared copy online. You can commit all flight long and sync up later.

3. You accidentally pushed an API key to a public repo, then deleted it in the next commit. Are you safe?

GitHub keeps every version ever, and automated scanners watch public code constantly. Treat a pushed key as compromised and replace it.
