Picture two interns on day one. You give both the same job: “Find me a flight to Miami under $200.”

Intern one leans back and says, “Have you tried comparing sites? Flying Tuesday is usually cheaper.” Technically true. Completely useless. Intern two opens a laptop, searches, finds nothing under budget, shifts the dates, widens the search, and comes back twenty minutes later with: “$187, Tuesday, nonstop. Want me to book it?”

You just watched that exact difference play out in the widget. The first button is a chatbot. The second is an agent.

An agent is an AI running in a loop: it takes a goal, thinks about a move, acts using a tool, observes what happened — and goes again until the job is done.

The loop is the whole idea

A chatbot’s life is one round: you talk, it talks, end of story. An agent’s life is a cycle — think → act → observe — and the magic lives in the third step. When the first search came back at $240, the agent didn’t shrug and report failure. It looked at the result, adjusted the plan, and tried again with flexible dates. Then again with a wider window. That’s not some extra intelligence bolted on top. That’s just the loop doing its job.

Notice it took three cycles in the demo. Real agents might take thirty. The loop keeps running until the goal is met, the agent decides it’s stuck, or it hits a limit you set — and good builders always set one.

Tools are the hands

“Act” doesn’t happen by magic. An agent acts through tools — and tools are mostly APIs, the order windows you met earlier. That search_flights() call in the widget is exactly that: an order placed at a flight-data window. Give the same loop more tools — calendar, email, your database — and it can do more jobs. Plug them in through MCP and you barely have to wire anything yourself.

Two more pieces complete the picture. The software wrapped around the model — the thing that actually runs the loop, calls the tools, and feeds results back in — is the harness. And if you want to build your own agent instead of renting one, you don’t start from zero: that’s what an Agents SDK is for.

When you want an agent — and when you don’t

A chatbot is the right tool when the answer is the product: explain this contract, draft this email, brainstorm ten names. Reach for an agent when the doing is the product — multi-step work with checkable results. Research this market and compile a doc. Fix this bug and run the tests. Find the flight and hold the seat. If you’d hand the task to an assistant rather than ask a friend for advice, it’s agent-shaped.

An AI that can act can also act wrong — at full speed, on repeat. That’s why builders cap the loop (max cycles, spending limits), shrink the tool menu to what the job needs, and keep a human approval step in front of anything irreversible. Give the intern a company card limit before sending them shopping.

What this unlocks for you

  • An inbox triager that reads, labels, drafts replies, and flags only the three emails that truly need you
  • A research agent that searches, cross-checks sources, and assembles a brief while you sleep
  • A coding agent that writes a fix, runs the tests, sees one fail, and patches its own work
  • An ops agent that watches a dashboard and files the ticket the moment something breaks

Same model in every case — Claude doesn’t get smarter between the two buttons you pressed. The difference is the loop around it and the tools in its hands.

A chatbot answers you. An agent comes back with the job done.

✓ Check yourself

1. You tell an AI: “Get me a table for four on Friday night.” Which response tells you it’s an agent, not a chatbot?

An agent acts through tools and reports results — including adjusting when the first try (7:00) fails. Advice, however good, is chatbot territory.

2. Mid-task, an agent’s first search returns zero flights under budget. What does a well-built agent do next?

The observe step exists so results feed the next think. Instant surrender or the same action on repeat means the loop isn’t actually looping.

3. Why do builders put guardrails — tool limits, spending caps, approval steps — on agents?

A chatbot’s worst output is a bad paragraph. An agent’s worst output is the wrong refund issued forty times. Guardrails scope the blast radius.
