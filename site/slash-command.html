Your best prompts, on speed dial

Everyone who works with AI has a few killer prompts — the summary format that’s exactly right, the review checklist that catches everything. And most people retype them. From memory. Slightly differently every time. Usually worse.

A slash command freezes the good version. The day you wrote it, you were careful: “exactly 3 bullets, under 20 words each, keep the numbers, end with a next action.” Every /summarize after that fires the careful version — including for the Friday-afternoon you who would otherwise type “sum this up pls.”

The character counter in the widget is the whole pitch: 10 keystrokes in, 240+ characters out. You’re not typing less because you’re lazy. You’re typing less because the thinking is already done.

Commands can take input, too. Type /translate-spanish see you Tuesday and everything after the command gets woven into the saved prompt — speed dial with a message attached. One shortcut, infinite variations, and the careful part never gets retyped.

One command, one team, one voice

Solo, slash commands save time. On a team, they create consistency. If five people summarize client calls five different ways, your call notes are chaos. Give everyone the same /call-notes command and the outputs line up — same sections, same tone, same fields — because everyone is firing the literally identical prompt. And when the prompt improves, you update one file and the whole team’s next /call-notes is upgraded with it.

It’s the same standardization play as a skill, one size smaller: a skill packages a whole procedure with steps and scripts, a slash command packages a prompt. They’re cousins — skills can be fired as slash commands too. And both are different from the system prompt, which is always on: a command only runs when you call it.

Where they live

In Claude Code, slash commands are just markdown files in a .claude/commands folder — one file per command, and the filename becomes the name. Create a file, type the prompt, done. The little builder in the widget was a toy version of exactly that. Type / in the CLI and the menu pops up: built-ins, your commands, your team’s, plus any installed skills.

Because they’re files, they travel with the project the same way CLAUDE.md does: clone the repo, inherit the commands. A new teammate types / on day one and sees how your team works.

The rule of three: the third time you type the same prompt, stop and save it as a command. Thirty seconds of saving beats a hundred retypings — and the saved version is the one you wrote when you were paying attention.
