SSH — Secure Shell — lets you sit at one computer and drive another one, through an encrypted tunnel.

"SSH into the box," decoded

When a builder says that, they mean: open a terminal — the CLI you already know — type ssh you@your-server, prove who you are, and from that moment every command you type runs on the distant machine. Your keystrokes go out encrypted, the output streams back encrypted, and nobody in between can read either. It's a screen-share for a computer that has no screen. And because you're only sending keystrokes and text — not video — it works fine over hotel Wi-Fi that can barely load a photo.

And that's the point: the machine running your site is a server in a warehouse somewhere — no monitor, no keyboard, no chair. SSH is the door. App crashed at 11 PM? Nobody drives to a data center. You SSH in from the couch, restart it, read a log or two, close the tunnel, and go back to your evening. This is also how a hands-on deploy works on a plain server: SSH in, pull the new code, restart.

Keys, not passwords

That "key check" in the handshake was real. SSH normally skips passwords in favor of a key pair: a public key that sits on the server like a lock, and a private key that stays on your laptop like the only key ever cut for it. Think of SSH keys as API keys for machines — and guard the private one at least as hard as anything in your env file. Whoever holds it is you, on every server that trusts it.

Never paste your private SSH key into a chat, an email, or a code repo — and never let an app "just borrow" it. If it ever leaks, don't debate: generate a new pair and remove the old lock from every server, immediately. A leaked private key isn't a vulnerability; it's a spare you, walking around with your wallet.

What this unlocks for you

  • Rescue from anywhere — restart a crashed app from a coffee shop, an airport, your phone
  • Live eyes — tail the server's logs in real time while users are on the site
  • Real maintenance — install updates, check disk space, move files, no plane ticket required
  • Agent hands — when Claude manages real infrastructure for you, SSH is often the tunnel it uses: same commands, same keys, same rules

Once you've felt the prompt change from samin@laptop to samin@server-tokyo, the phrase "I'm on the server" stops being mysterious. You're not on anything. You're in your kitchen, holding the walkie-talkie, and Tokyo is doing exactly what you say.

The server has no screen. SSH is how you become its keyboard.
