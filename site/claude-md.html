TLDRHouse rules your AI reads before it starts.

A good house-sitter doesn’t guess. You leave a laminated sheet on the fridge: the cat eats at 7, bins go out Tuesday, never open the door for the “gas company,” wifi password is on the router. Any sitter, any week — the house runs the same way.

CLAUDE.md is that sheet, for your project.

CLAUDE.md is a plain text file in your project folder that Claude Code reads automatically at the start of every session — your house rules, written down once.

It’s just a file. That’s the best part

No settings screen, no special syntax to learn. It’s markdown — plain text with light formatting — sitting in your project folder, named CLAUDE.md. Open it, write your rules in plain English, save. From then on, every new session begins with Claude reading the sheet before touching anything.

You saw the effect in the widget. “Add a save button” is an ambiguous request, and Claude filled the gaps differently depending on which rules were on. Tests-first on? The test file came before the button. Brand rule on? Out comes #d97757. Every rule off? You got generic JavaScript pushed straight to main — the house-sitter who never got a sheet, improvising in your kitchen.

What belongs on the sheet

Short, specific, checkable entries:

  • Conventions: “Use TypeScript.” “Styles live in styles.css, never inline.”
  • Commands: “Run npm test before any commit.” “Start the dev server with npm run dev.”
  • Never-dos: “Never push to main.” “Never edit the /legacy folder.”
  • Context: “This is a Next.js app. The API lives in /server.”

Notice every rule is checkable. “Write good code” is a vibe. “Tests first” is a rule. Vibes get ignored; rules get followed.

Starting one takes two minutes: create the file, write the five rules you already know you care about. The rest accumulates the natural way — every time Claude does something that makes you wince, that wince becomes a line in the file. Within a week, the sheet knows your project better than your last contractor did.

Project memory vs. chat memory

Why not just tell Claude your preferences in chat? Because chat memory dies. A session is one conversation — close it, and what you said is gone. Tomorrow’s session starts blank, and you’re explaining the brand color again, like briefing a new sitter every single morning.

CLAUDE.md flips that. The file lives with the project, so the rules survive every session, every teammate, every new laptop. It’s also a close cousin of the system prompt — both are standing orders an AI follows. The difference is ownership: the system prompt is wired into the product by whoever built it. CLAUDE.md is yours — per project, editable any time, saved right next to your code. And for repeatable procedures rather than standing rules, there’s a third tool: the skill, coming up.

Keep it short. CLAUDE.md rides into the context window at the start of every session — a bloated file taxes Claude’s attention (and your token bill) on every single request. Ten sharp rules beat three pages of philosophy. Prune it like a packing list.

What this unlocks for you
