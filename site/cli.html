CLI

TLDRTalk straight to the computer. No clicking.

There are two ways to order food. You can point at the picture on the menu and hope the kitchen figures it out. Or you can walk up and say exactly what you want, in the kitchen's own shorthand, and get exactly that.

Pointing at pictures is the GUI — the graphical interface of windows, icons, and buttons you've used your whole life. The second way is the CLI.

The CLI — command line interface — is where you type instructions straight to the computer instead of clicking through screens.

It's that black window from the hacker movies. And you're about to use one: ls to look around, open to read a file, deploy to ship a site. Type a thing, get a thing. That's the entire interface. (If you tried make coffee, you also met the terminal's personality: literal, honest, one job per command.)

One line beats six clicks

Why would anyone give up buttons? Three reasons builders never go back:

  • Precision. A command says one exact thing. No "wait, which of these three gear icons is settings?"
  • Repeatability. A command is text, so you can save it, paste it, run it a hundred times, or schedule it every morning with a cron job. "Click the third menu, then the gray button" is not a recipe a computer can rerun. deploy is.
  • Power. A GUI shows you the options somebody decided to draw buttons for. The CLI exposes everything the tool can do.

You saw the strip under the widget: the GUI route to publishing a site is six clicks across three screens. The CLI route is one word. Now imagine doing that every day, forever.

This is where Claude Code lives

Here's why this lesson matters more now than ever: AI agents are CLI-native. A language model's whole world is text in, text out — and a terminal is exactly that. No pixels to interpret, no buttons to hunt for.

So when Claude Code builds something for you, it's sitting in a terminal typing commands, just like you just did: listing files, installing packages, running tests, deploying. Every tool with a CLI is a tool your AI can drive at full speed. There is a slower path where AI literally moves a mouse around the screen — that's computer use, later in the course — but text is its native language.

One more thing: the machines your projects eventually run on — servers in a data center — don't have screens at all. Out there, the terminal isn't a preference. It's the only door in. You'll walk through it in the SSH lesson.

A terminal does what you say, not what you mean — and plenty of commands have no undo and no "are you sure?" Make this a habit: paste any unfamiliar command to Claude first and ask "what does this do?" Ten seconds of checking, zero days of regret.

✓ Check yourself

1. Claude tells you: "run npm install in your project folder." Where does that happen?

Commands run in the terminal. The address bar finds websites; the terminal runs instructions on your machine.

2. You mistype a command and the terminal says "command not found." What actually happened?

Terminals don't guess — you saw this in the widget. Wrong text means no action, which is exactly what makes the right text so powerful.

3. Why do AI agents like Claude Code do their work in a terminal instead of clicking around the screen?

AI can click screens too — that's computer use, coming in Level 3 — but text commands are faster, more precise, and exactly how a language model already thinks.
