Deploy

TLDRPush your work live for the world to use.

Picture a restaurant kitchen during dinner service. The chef finishes a dish, wipes the rim, and slides the plate up onto the pass — that steel counter where the kitchen ends and the dining room begins. Until that slide, the dish is private: taste it, fix it, burn it and start over, nobody will ever know. The second it hits the pass, it's real. A customer is about to eat it.

Building software works exactly like this. The pass even has a name.

Deploying is sliding your work up onto the pass — copying your project from your computer onto a server so real people can actually use it.

Everything you build starts on localhost, your private kitchen. The site runs, the buttons click — but only on your machine. Text the link to a friend and it dies, because there's nothing at their end to reach. Deploying puts a copy of your project on a server — a computer that never closes — and that's the exact moment a project becomes a product.

Try it — the ship-it pipeline

What actually happened when you hit 🚀

The pipeline you just ran is the real sequence, minus a thousand details you don't need yet:

  • Build — your files get bundled and squashed into a tidy package that servers can hand out fast.
  • Test — automatic checks run first. If one fails, the pipeline stops cold, and the broken version never reaches a single visitor.
  • Upload — the package is copied onto the server.
  • Live — the server starts serving the new version to anyone on Earth. That's it. You shipped.

If you've ever asked Claude to build you a landing page and thought "okay, but how does this get on the internet?" — this is the answer. Modern platforms run that whole chain for you in about a minute. You click deploy, you're live. The hard part isn't the mechanics anymore. It's the nerve.

The part nobody tells beginners: it's reversible

Deploy sounds permanent, which is why it feels scary. It isn't. You proved that yourself: you shipped v1, shipped v2, then rolled back to v1 in one click. The server kept v1's files the whole time, so "rollback" just means "point everyone at the old version again." Instant. No drama.

This is the same save-point superpower you get from Git: every deploy is a version you can return to. Once rollback clicks in your head, deploying stops feeling like jumping off a cliff and starts feeling like walking through a doorway you can step back through.

One warning before you ship anything: your code travels when you deploy, and secrets must not travel with it. API keys belong in a hidden env file that stays out of the package — never pasted into the code itself. Bots scan fresh deploys and public code for leaked keys around the clock.

What this unlocks for you

  • Take the page Claude built you this morning and put it on a real URL by lunch
  • Fix a typo on your live site and ship the fix in under a minute
  • Ship on a Friday evening without sweating, because rollback exists
  • Stop emailing screenshots of localhost and start sending actual links

Your project on localhost is a rehearsal. Deploying is opening night — except in this theater, if the show goes badly, one button puts last night's performance back on stage while you fix things calmly backstage.

Don't polish forever in the kitchen. The pass is right there — ship the plate.

✓ Check yourself

1. You built a booking page with Claude and it looks perfect at localhost:3000. You text the link to a client — "it doesn't load." What happened?

Localhost is your private kitchen. Until you deploy, there is no public copy anywhere for anyone else to load.

2. You deploy v2 and the homepage comes up blank — with 200 visitors on the site. Best first move?

Rollback is instant because the server kept the old version's files. Visitors get working v1 back in seconds while you debug with zero pressure.

3. Mid-deploy, the test stage fails: 1 of 3 checks broken, pipeline stops. What did that just do for you?

The pipeline halts on a failed test, and the live site keeps serving the last good version. Broken code never makes it to the pass.
